# **SMV-01 | Logical issue in the layer2 rewards distribution**

| **카테고리** | **심각도** | **위치** | **상태** |
| --- | --- | --- | --- |
| **Logical-issue** | **토론** | **SeigManagerV1_3.sol#L667-L669: L667-L669** | **Pending** |

### **Description**

In the case of `_isSenderOperator || oldLayer2Info.layer2Tvl > curLayer2Tvl`, the `layer2Seigs` will be distributed to the l2 via the l2 manager function `updateSeigniorage`.

```solidity
  ILayer2Manager(_layer2Manager).updateSeigniorage(rollupConfig, layer2Seigs);

```

However, in the `else` case, which means `!_isSenderOperator && oldLayer2Info.layer2Tvl < curLayer2Tvl && _lastCommitBlock[msg.sender] == 0`, the initialDebt will be updated according to the oldLayer2Info. The conditions mean the caller is not the operator and the `_updateSeigniorage` is never called before.

As a result, the accumulated rewards will be cleared by a non-operator.

```solidity
  else if (_lastCommitBlock[msg.sender] == 0) {
              newLayer2Info.initialDebt = (l2RewardPerUint * oldLayer2Info.layer2Tvl) / 1e18;
  }

```

We would like to confirm with the client whether this is by design.

### **Recommendation**

Recommend to review the code and ensure there is no logical inconsistency.

When the update seigniorage of the layer2 is first executed, this code is executed. [code link](https://github.com/tokamak-network/ton-staking-v2/blob/75db2c10cc0e6d97f16323fe97f6d693388891fb/contracts/stake/managers/SeigManagerV1_3.sol#L667-L669)

```

else if (_lastCommitBlock[msg.sender] == 0) {

newLayer2Info.initialDebt = (l2RewardPerUint * oldLayer2Info.layer2Tvl) / 1e18;

}

```

[In the unSettledReward function ](https://github.com/tokamak-network/ton-staking-v2/blob/75db2c10cc0e6d97f16323fe97f6d693388891fb/contracts/stake/managers/SeigManagerV1_3.sol#L374-L378), the seigniorage that needs to be settled is calculated,

```

function unSettledReward(address layer2) public view returns (uint256 amount) {

Layer2Reward memory layer2Info = layer2RewardInfo[layer2];

if (layer2Info.layer2Tvl != 0)

amount = l2RewardPerUint * (layer2Info.layer2Tvl / 1e18) - layer2Info.initialDebt;

}

```

The **l2RewardPerUint** used in this function is the seigniorage reflected since layer2StartBlock.

So since the start block (update seigniorage of L2) is larger than layer2StartBlock, **the seigniorage corresponding to the seigniorage in between the start block and layer2StartBlock should be stored as initialDebt and become the amount that must be deducted later.**
