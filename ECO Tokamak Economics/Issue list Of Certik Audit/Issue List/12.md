**이슈내용** 

```solidity
modifier onlyMemberContract() {
    address candidate = ICandidate(msg.sender).candidate();
    require(isMember(candidate), "StorageStateCommittee: not a member");
    _;
}
```

onlyMemberContract modifier에서의 검사만 했을때는 candidate함수를 통과시켜서 악의적으로 공격할 수 있습니다.

**내용 확인 **

해당 검사만 했을때는 문제 있는 것이 맞습니다.

하지만 추가적으로

```solidity
require(
    candidateInfo.candidateContract == msg.sender,
    "DAOCommittee: invalid candidate contract"
);
```

이 것을 검사하여서 해당 modifier로 문제있는 함수는 없습니다.

해당 modifier는 현재 retireMember()함수만 사용하고 있습니다.

**조치 내용**

해당 modifier는 문제가 없으나 retireMember()함수를 확인했을때 같은 address를 2번 선언하여서 gasFee의 낭비를 확인했습니다.

그래서 onlyMemberContract modifier는 사용하지않고 

```solidity
require(
    candidateInfo.memberJoinedTime > 0,
    "DAOCommittee: not a member"
);
```

를 retireMember()함수에 추가하려 합니다.
