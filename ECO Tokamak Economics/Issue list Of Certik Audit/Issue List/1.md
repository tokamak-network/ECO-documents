# **SMV-04 | Inconsistent calculation of operatorSeigs**

| **카테고리** | **심각도** | **위치** | **상태** |
| --- | --- | --- | --- |
| **Logical-issue** | **토론** | **SeigManagerV1_3.sol#L534-L537: L534-L537** | **Pending** |

### **Description**

When the `isCommissionRateNegative_` is negative(false), the operatorSeigs calculation is like: `operatorSeigs = rmul(seigs, commissionRate);`

When the `isCommissionRateNegative_` is positive(true),the operatorSeigs calculation is like below:

```solidity
// ɑ: insufficient seig for operator
       operatorSeigs = rmul(
            rmul(seigs, operatorRate), // seigs for operator
            commissionRate
        );
// β:
        uint256 delegatorSeigs = operatorRate == RAY
            ? operatorSeigs
            : rdiv(operatorSeigs, RAY - operatorRate);
// 𝜸:
        operatorSeigs = operatorRate == RAY
            ? operatorSeigs
            : operatorSeigs + rmul(delegatorSeigs, operatorRate);

```

We would like to confirm with the client for below 2 questions:

1. Why there is a difference in the calculations of operatorSeigs with different `isCommissionRateNegative_`.
1. Can you provide some explanations about the positive case calculation formulas.
