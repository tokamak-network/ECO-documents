# **SMV-03 | Possibility of the operator receiving or losing tokens twice**

### **Description**

In the function `_calcSeigsDistribution`, if `isCommissionRateNegative_` is negative(false), the `operatorSeigs` will be deduct from the `nextTotalSupply`. If `isCommissionRateNegative_` is positive(true), the `delegatorSeigs` will be added to the `nextTotalSupply`. The following up logic will `burnFrom` or `mint` to the operator with the relative amount. The minting or burning of `operatorSeigs` causes adjustment to the operator's coinage balance.

```solidity
     if (operatorSeigs != 0) {
            if (isCommissionRateNegative_) {
                // TODO: adjust arithmetic error
                // burn by 𝜸
                coinage.burnFrom(operator, operatorSeigs);
            } else {
                coinage.mint(operator, operatorSeigs);//commision
            }
        }

```

Additionally, the `operatorSeigs` is added to or deducted from the `nextTotalSupply`, it directly impacts the `coinage.setFactor()` and causes a second adjustment to the coinage balance with the operatorSeigs relative amount.

```solidity
  nextTotalSupply -= operatorSeigs;
 ...
  nextTotalSupply += delegatorSeigs;

```

We would like to confirm with the client whether this is by design.
