### **Description**

In solidity, function signatures (or selectors) are derived from the function's name and input types. If two different implementations (SeigManagerV1_2 and SeigManagerV1_3) have the same function signature, it means that calls to these functions will resolve to the same selector. This can lead to unexpected behavior, as the proxy will not distinguish between the two versions and might invoke the wrong implementation or cause contract logic issues.

The above-mentioned 5 functions in the contract `SeigManagerV1_3` share the same function signatures with the contract `SeigManagerV1_2`. According to the deploy script, the `SeigManagerV1_3` contract will be set as the aliveImplementation of `ProxySeigManager`.
