## Introduction

Tokamak Network is beginning an important phase of transitioning our existing Tokamak Staking service to a community-driven model to align with our vision of decentralization and community-centric ecosystems. The staking service, previously managed directly by Tokamak Network, will now be hosted and operated by community members.

## Introducing the Community Version

### Innovation Towards True Decentralization

The **Community Version** is a new approach developed to fundamentally solve the centralization issues of the existing staking service.

**Limitations of the Existing Service:**
The existing Tokamak Staking service was completely dependent on backend services such as subgraphs and APIs. Without this backend infrastructure, service operation was impossible, resulting in the service inevitably operating in a centralized state.

**Innovation of the Community Version:**
The community version has completely eliminated this backend dependency. No longer relying on centralized backend infrastructure, **anyone can now independently operate the staking service**.

**Core Features:**

- **Backend Independence**: No need for centralized backend services like subgraphs and APIs
- **Complete Autonomy**: Individual users or communities can operate services independently
- **True Decentralization**: No longer dependent on a single operating entity
- **Identical Functionality**: Provides essentially the same staking features as the existing service

## Background and Conditions of Transition

### Phased Transition Strategy

The community version fundamentally provides the same functionality as the existing staking service, but with the key difference of independence from backend infrastructure.

**Initial Support Phase:**
Tokamak Network will initially officially host the community version, providing users with sufficient time to familiarize themselves with the new environment.

**Complete Transition Phase:**
After a certain period following the closure of official hosting, the service will transition fully to community or individual user hosting, including local environments.

## Staged Restriction Timeline

To minimize confusion and ensure a smooth transition, the following schedule will be implemented:

### Major Transition Process

|  | Content | Notice |
| --- | --- | --- |
| 21st July | Open Community version (hosting both version) | 1 blog + 1 x posting |
| 28th July | Restriction of unstake and withdraw button(only available at community version) | x posting |
| 4th Aug | Restriction of stake button | x posting |
| 11th Aug | Shutdown the simple staking | x posting  + documentation |
| 18th Aug | Shutdown hosted community version | x posting |

**Week 1 - Introduction Phase:**

- Official opening of the community version
- Adding a community version link or banner to the existing simple staking service

**Weeks 2-3 - Feature Restriction Phase:**

- Gradual restriction of core functions in the existing service
- Natural guidance of users to the community version
- Providing appropriate warning messages and tooltips at each restriction stage

**Week 4 - Service Shutdown:**

- Complete shutdown of the existing simple staking service
- Release of comprehensive documentation, including the Etherscan guide
- Providing guides for users to directly interact with contracts

**Week 6 - Complete Decentralization:**

- End of official hosting by Tokamak Network
- Start of complete autonomous community operation
