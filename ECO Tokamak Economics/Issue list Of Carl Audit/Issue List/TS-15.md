## Feedback

CnadidateAddOn 컨트랙이 등록되는 과정을 담은 설명입니다. 

[ton-staking-v2/docs/en/ton-staking-v2.md at v2.5-audit-request · tokamak-network/ton-staking-v2](https://github.com/tokamak-network/ton-staking-v2/blob/v2.5-audit-request/docs/en/ton-staking-v2.md#register-candidateaddon)

그림을 보시면 CandidateAddOnFactory에 의해서[ CnadidateAddOnProxy가 배포](https://github.com/tokamak-network/ton-staking-v2/blob/01e198130b178757dd194bd8726a1ab678fca167/contracts/dao/factory/CandidateAddOnFactory.sol#L96)되고, 로직이 설정되고, 초기화가 진행됩니다. 

CnadidateAddOn 로직과 초기화때 사용되는 ton, wton의 주소는 CandidateAddOnFactory 의 스토리지에 미리 설정된 값입니다. 

우리는 초기 배포후, ton 과 wton에 대한 [초기값 설정](https://github.com/tokamak-network/ton-staking-v2/blob/01e198130b178757dd194bd8726a1ab678fca167/contracts/dao/factory/CandidateAddOnFactory.sol#L51C5-L76)을 올바르게 하고 있습니다.   

```json
   function setAddress(
        address _depositManager,
        address _daoCommittee,
        address _candidateAddOnImp,
        address _ton,
        address _wton,
        address _onDemandL1BridgeRegistry
    ) external onlyOwner {
        require(
            _ton != address(0) && _wton != address(0) &&
            _depositManager != address(0) && _daoCommittee != address(0) && _candidateAddOnImp != address(0)
            && _onDemandL1BridgeRegistry != address(0) , "zero");

        require(
            ton != _ton || wton != _wton ||
            depositManager != _depositManager || daoCommittee != _daoCommittee || candidateAddOnImp != _candidateAddOnImp
            || onDemandL1BridgeRegistry != _onDemandL1BridgeRegistry , "same");

        depositManager = _depositManager;
        daoCommittee = _daoCommittee;
        candidateAddOnImp = _candidateAddOnImp;
        ton = _ton;
        wton = _wton;

        onDemandL1BridgeRegistry = _onDemandL1BridgeRegistry;
    }
```
