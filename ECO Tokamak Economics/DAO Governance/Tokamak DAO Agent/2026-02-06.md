## TON과 WTON의 Uniswap 거래 가능 여부

### ✅ **기술적으로는 가능합니다**

TON과 WTON 모두 **표준 ERC20 토큰**이므로, 기술적으로는 Uniswap V2/V3에서 거래가 가능합니다.

  - **TON**: `0x2be5e8c109e2197D077D13A82dAead6a9b3433C5`
    - 표준 ERC20Mintable + ERC20Detailed 구현
    - `transfer`, `transferFrom`, `approve` 등 표준 함수 지원
  - **WTON**: `0xc4A11aaf6ea915Ed7Ac194161d2fC9384F15bff2`
    - ERC20 + TON/WTON 스왑 기능 내장
    - 표준 ERC20 인터페이스 완벽 지원

### 🔍 **하지만 주의사항이 있습니다**

### 1. **TON의 transfer 제약**

TON 컨트랙트 코드를 보면 특별한 제약이 없는 표준 ERC20이지만, **SeigToken**을 상속하고 있습니다:

```solidity
contract TON is Ownable, ERC20Mintable, ERC20Detailed, SeigToken
```

SeigToken의 `transferFrom` 함수에는 특별한 제약이 있습니다:

```solidity
function transferFrom(address sender, address recipient, uint256 amount) public returns (bool) {
    require(msg.sender == sender || msg.sender == recipient,
            "SeigToken: only sender or recipient can transfer");
    return super.transferFrom(sender, recipient, amount);
}
```

이는 **Uniswap 라우터가 토큰을 전송할 때 문제가 될 수 있습니다**. Uniswap 라우터는 제3자로서 `transferFrom`을 호출하므로, 이 조건을 만족하지 못할 수 있습니다.

### 2. **더 나은 방법: Tokamak 내부 스왑 사용**

Tokamak Network는 자체적으로 **TON ↔ WTON 스왑**을 제공합니다:

```solidity
// WTON 컨트랙트의 스왑 함수들
swapFromTON(uint256 tonAmount)          // TON → WTON
swapToTON(uint256 wtonAmount)           // WTON → TON
swapFromTONAndTransfer(address to, uint256 tonAmount)
swapToTONAndTransfer(address to, uint256 wtonAmount)
```

**장점:**

  - ✅ 수수료 없음 (1:1 비율로 정확히 교환)
  - ✅ 가스비가 저렴
  - ✅ 슬리피지 없음
  - ✅ 유동성 문제 없음

### 💡 **실제 사용 시나리오**

```
1. TON ↔ WTON 교환이 필요한 경우
   → WTON 컨트랙트의 swap 함수 사용 (추천)

2. TON/WTON ↔ ETH/다른 토큰 교환
   → Uniswap 사용 가능 (유동성 풀이 있다면)
   → 단, TON의 transferFrom 제약 때문에 문제가 있을 수 있음

3. 스테이킹을 위해 TON을 WTON으로 변환
   → TON.approveAndCall()을 통한 자동 변환 + 스테이킹
```

### 📊 **요약**

| 교환 페어 | Uniswap 사용 | 추천 방법 | |----------|-------------|----------| | TON ↔ WTON | ⚠️ 제약 있음 | WTON 컨트랙트 swap 함수 | | TON ↔ ETH | ✅ 가능 (풀 있다면) | Uniswap (유동성 확인 필요) | | WTON ↔ ETH | ✅ 가능 (풀 있다면) | Uniswap (유동성 확인 필요) |

### 🎯 **결론**

**TON과 WTON은 기술적으로 Uniswap에서 거래 가능하지만**, TON의 `transferFrom` 제약 때문에 실제로는 문제가 있을 수 있습니다.

**가장 안전하고 효율적인 방법은:**

  - TON ↔ WTON 교환: **WTON 컨트랙트의 swap 함수 직접 사용**
  - 다른 토큰과의 교환: **Uniswap 사용 (WTON 사용 권장)**

Tokamak Network는 내부적으로 최적화된 스왑 메커니즘을 제공하므로, 외부 DEX를 사용하는 것보다 네이티브 swap 기능을 사용하는 것이 더 좋습니다! 🚀
