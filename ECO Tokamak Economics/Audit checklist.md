# 1. Contract Process for Auditing 

(1) 오딧 견적 요청을 위한 문서 준비 Audit request document

(2) 오딧 계약 절차 (업체인 경우) Audit contract procedure (for companies)

(3) 오딧 계약 절차 (개인인 경우)  Audit contract procedure (for individuals)

(4) 계약의 선수금 (또는 중간금, 잔금) 집행 

(5) 오딧 작업 진행  Audit Progress Document

# 2. Checklist before contract 

- [ ] 계약 조건 검토 ([참고](/1bbd96a400a38043a479c3a64e5b5c49#1d2d96a400a380a5a7b4c6e1ddffbf37))
- [ ] 산출물 검토
- [ ] 계약 관련 사항 공유 채널 생성

# 3. Checklist 계약후 

- [ ] 계약 금액(선금) 및 주소 확인 후, 선금 집행 
- [ ] 오딧 진행 문서 작성 하였는가? 
- [ ]  계약서에 기재된 일정을 토카막 캘린더에 등록하였습니까?  오딧 시작일,  오딧 종료일,  오딧 1차 보고서 받는 날짜, 오딧 1차 보고서에 따른 피드백 마감날짜, 선수금 지불 날짜, 잔금 지불날짜 등을 캘린더에 기재합니다.
- [ ] 업체별 커뮤니케이션 채널 생성

# 4. Checklist when got first report

- [ ] 계약에 따라 1차 보고서 제출일이 지났는데, 1차 보고서가 제출되지 않은 경우, 오디터에게 연락하여, 보고서 제출을 요청합니다.  
- [ ] 계약 사항에 따라 중도금지급이 있는지 확인합니다.
- [ ] 1차 보고서가 나오면 채널에 공지하여 공유합니다. 

# 5. Checklist during reflect feedback

- [ ] 1차 보고서에 대한 피드백 제출 일정을 체크합니다.
- [ ] 피드백 보고서도 채널에 공지하여 공유합니다. 
- [ ] 업체별 피드백 제출 이후의 프로세스와 일정을 확인하여 채널에 공유합니다. 

# 6. Checklist before final report

- [ ] 최종 산출물 확인
- [ ] 외부 공개 확인
- [ ] 계약 금액(잔금) 및 주소 확인

## 계약시 검토 내용    

### 결제 수단  

- 가상자산 선호 (USDT, USDC 등)  
- 달러 결제 가능 ( 재단의 상황에 따라 다르므로, 재단의 확인이 필요함) 

### 결제 일정

- 중요 포인트 : **최종 결제는 최종보고서를 받고 난뒤 지불을 해야하고 그렇지 않다면 계약진행을 하지 않는 것이 바람직합니다. **
- 아래와 같은 방법의 분할 결제를 추천합니다. 
  - 최초 진행시 50%, 최종 보고서를 받고 남은 잔금 50% 지급
  - 최초 진행시 30%, 1차보고서를 받고 30%, 최종보고서를 받고 40% 지급

### 연체료 및 패널티 확인

- 연체료 및 패널티가 있을때 해당 내용과 금액을 재단 관계자, 개발 관련 담당자가 모두 공유를 해야 하고, 업무 문서에 기재해야 합니다.  
- 패널티의 내용이 있다면 구체적으로 명시하는 것이 바람직합니다.  불리한 내용이 있는 경우, 계약을 하지 않는것이 좋습니다. 
  - Audit에서 나온 취약점이 많으면  추가금 확인
    - 취약점이 gas효율 등 코드개선 수준도 포함인지 아닌지 또는 치명적인 자금 동결이나 자금 손실과 같은 심각도에 한한것인지 등 확인

- Zena
