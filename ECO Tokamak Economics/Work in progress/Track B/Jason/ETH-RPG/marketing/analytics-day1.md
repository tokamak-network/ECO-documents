## 3. 퍼널 분석 (보정)

```
외부 Landing (~17명)
  → Address Input (2~3회)      = ~15% 전환
    → Card Generated (2건)     = ~80% 전환
      → Battle (1건)           = 50%
        → Ranking (1건)        = 100%
```

| 단계 | 전환율 | 평가 |
|------|--------|------|
| **Landing → Input** | **~15%** | **최대 병목.** 17명 중 2-3명만 주소 입력 시도 |
| Input → Card | ~80% | 양호. empty_wallet 에러 1건 외 성공 |
| Card → Battle | ~50% | 양호 |
| Battle → Ranking | 100% | 양호 |

**핵심 병목: Landing → Input (15%)**
- PH에서 15명이 왔지만 1명만 카드 생성
- 주소를 직접 입력해야 하는 허들이 높음
- "vitalik.eth 원클릭 데모" 같은 즉시 체험 기능이 전환을 올릴 수 있음

## 8. 핵심 인사이트

### 긍정적
1. **제품은 작동한다** — 외부 유저가 풀 퍼널(카드→배틀→랭킹) 경험
2. **리텐션 신호** — 1명이 ranking 3번 재방문 (랭킹이 재방문 동기 생성)
3. **에러 복구** — empty_wallet 에러 후 유저가 재시도하여 성공
4. **Input→Card 전환율 높음 (~80%)** — 주소를 넣으면 결과까지 잘 도달
5. **안정적 운영** — 에러 1건뿐, 다운타임 없음

### 부정적
1. **Landing → Input 전환율 15%** — 최대 병목. 17명 중 2-3명만 시도
2. **PH 15명 방문 → 1건 전환 (6.7%)** — 구경만 하고 이탈
3. **절대 트래픽 부족** — 외부 유저 ~17명
4. **6개 채널 중 실질 작동 2개** (PH, Dev.to만)
5. **Twitter/Farcaster/HN/Reddit 모두 신규 계정 제한**

### 병목 2가지
1. **도달(Reach)**: 신규 계정으로 플랫폼 도달 불가 → 인적 네트워크로 피봇
2. **Landing 전환**: 방문해도 주소 입력을 안 함 → UX 개선 필요 (원클릭 데모 등)
