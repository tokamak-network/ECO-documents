# ETH-RPG Fun Factor Review

> Volkov 심판 결과 정리 | 2026-02-20 | Score: 5.8/10 (REJECT)

---

## 핵심 진단

**"메커니즘은 있는데, 영혼이 없다."**

배틀 엔진은 체계적으로 설계되었으나, 유저가 접하는 표면(카드, 로딩, 스탯 표시)에서 RPG 재미를 느낄 수 없다. 가장 흥미로운 데이터(크립토 이벤트, 스킬, 상성)가 코드 내부에 묻혀 유저에게 노출되지 않는다.

## 감점 사항 상세

### P0 — 구조적 결함 (Critical)

#### 1. 크립토 이벤트 타임라인 미노출

- **현상**: `crypto-events.ts`에 12개 이벤트가 RPG 용어로 번역되어 있음 ("달의 왕국 몰락", "대합병의 의식" 등)
- **문제**: AI 로어 생성과 LUCK 계산에만 내부 사용. 유저는 자기 지갑이 어떤 사건을 겪었는지 볼 수 없음
- **영향**: 서비스 최고의 스토리텔링 소재가 완전히 묻혀 있음
- **제안**: 카드 뒷면 또는 결과 페이지에 "생존한 사건" 타임라인 시각화 추가

#### 2. 배틀 스킬/패시브가 카드에 미표시

- **현상**: 8개 고유 스킬(NFT Snipe, Arbitrage, Ancient Spell 등)과 패시브가 배틀 엔진에만 존재
- **문제**: 캐릭터 카드에 스킬명, 패시브, 상성 정보가 전혀 없음. 카드 뒷면은 로어 텍스트만 반복
- **영향**: 유저가 자기 캐릭터의 고유 능력을 인지할 수 없음. 배틀 모드 유입 동기 약화
- **제안**: 카드 뒷면 재설계 — 스킬명 + 패시브 + 상성 약도 포함. TCG 카드처럼 능력 박스 배치

---

### P1 — 논리적 허점 (High)

#### 3. 폴백 로어 2개/클래스 — 반복 불가피

- **현상**: AI API 4단계 캐스케이드 실패 시 클래스당 short 2개 + long 2개 템플릿만 존재
- **문제**: 같은 클래스 유저 수백 명이 동일 로어를 볼 가능성
- **제안**: 클래스당 최소 10개로 확장, 레벨 구간별 변형 추가 (Lv1-20, 21-40, 41-60)

#### 4. 스탯에 맥락/설명 전무

- **현상**: STR 312, INT 280 같은 숫자만 표시. `STAT_MAX_VALUES` 코드에 있으나 UI 미노출
- **문제**: 유저가 자기 스탯이 높은지 낮은지, 왜 그 수치인지 알 수 없음
- **제안**:
  - 스탯별 한 줄 플레이버 텍스트 (예: "STR: DEX 스왑의 힘이 깃든 근력")
  - 임계값 라벨 (>400 = "전설적", >300 = "영웅급")
  - 스탯 클릭 시 산출 근거 툴팁 ("DEX Swap 450회 + Bridge 23회에서 산출")

---

### P2 — 실행 미비 (Medium)

#### 5. 업적 시스템 피드백 루프 부재

- **현상**: 16개 업적, 4티어, 실제 크립토 이벤트 기반 — 구조는 양호
- **문제**: 정적 뱃지만 존재. 해금 애니메이션, 진행도, 히든 업적, 칭호 연동 없음
- **제안**:
  - 진행도 표시 ("브릿지 7/10회 — Bridge Runner까지 3회!")
  - 업적 기반 칭호 생성 ("대합병의 증인, 달의 왕국 생존자")
  - 해금 시 시각적 피드백 (빛 효과, 사운드 옵션)

#### 6. 로딩 화면에 RPG 플레이버 없음

- **현상**: "트랜잭션 분석 중...", "스탯 계산 중..." — 4개 기술적 단계만 표시
- **문제**: 대기 시간이 이탈 구간인데 재미 요소가 없음
- **제안**:
  - 20개+ RPG 메시지 랜덤 ("오라클에게 자문 중...", "고대 문서를 해독하는 중...")
  - 크립토 트리비아 팁 ("이더리움 최초의 지갑은 2015년에 생성되었습니다")
  - 클래스 실루엣 티저 애니메이션

---

### P3 — 표현 개선 (Low)

#### 7. 클래스 설명이 기술 라벨 수준

- **현상**: 랜딩 페이지에 "Hunter — NFT ratio >= 25%" 식 표기
- **문제**: RPG 분위기 제로. `ClassResult` 타입에 `description` 필드 자체가 없음
- **제안**: 클래스별 모토 + 2줄 배경 스토리 추가
  - 예: "Hunter — 잊혀진 유물을 추적하는 자. 어떤 경매장도 이 사냥꾼의 손길을 벗어나지 못한다."

#### 8. OG/카드 이미지에 장식 없음

- **현상**: 순수 데이터 표시. 배경 패턴, 장식 프레임, 티어별 테두리 없음
- **문제**: 소셜 피드에서 "게임 카드"가 아닌 "대시보드 스크린샷"으로 보임
- **제안**:
  - 전투력 티어별 카드 프레임 (브론즈/실버/골드/다이아)
  - 클래스별 미묘한 배경 패턴 (마법진, 검무늬, 방패 문양 등)

---

### P1 — 설계/아키텍처

#### 9. 배틀 OG 이미지 미구현

- **현상**: `battle-pipeline.ts`에서 배틀 OG URL 생성하지만 `/api/og/battle/` 라우트 미존재
- **문제**: 배틀 결과 공유 시 기본 OG 이미지가 나옴. 바이럴 핵심 누락
- **제안**: 두 캐릭터 대결 OG 이미지 구현 (1200x630, 양쪽 스프라이트 + VS + 승자 하이라이트)

#### 10. 재미 콘텐츠가 코드에 결합

- **현상**: 이벤트 번역, 배틀 내러티브, 클래스 기준이 모두 TS 파일에 하드코딩
- **문제**: 콘텐츠 추가/수정에 개발자 필요. 기획자/작가 독립 작업 불가
- **제안**: 콘텐츠 레이어 분리 (JSON/YAML 파일, 향후 CMS 연동 가능 구조)

## 강점 (감점하지 않은 부분)

- **배틀 엔진**: 8 스킬 + 8 패시브 + 상성 2링 + 결정론적 PRNG + 크립토 테마 내러티브 — 체계적 게임 설계
- **픽셀 스프라이트**: 8클래스 x 6티어 = 48개 변형, 스탯 반응형 시각 이펙트 (HP 심장박동, MP 아케인, LUCK 반짝임)
- **시즌 시스템**: "Genesis Season", "Shattered Realms" 등 테마 이름
- **Famous Duels**: "Vitalik vs Uniswap" 등 유명 지갑 대결 — 발견성 우수
- **카드 플립**: 3D CSS 퍼스펙티브 회전 인터랙션
- **공유 텍스트**: "내 지갑 전투력 42,150... 너는?" — 바이럴 훅
