# ETH-RPG 리텐션 기능 제안 리뷰

> 날짜: 2026-03-03
> 목적: 서비스 내 "돌아올 이유"가 없는 핵심 문제 해결

---

## 핵심 진단

현재 ETH-RPG의 리텐션 실패 원인은 3가지 부재:

| 부재 | 의미 | 예시 |
|------|------|------|
| **변동성** | 매일 같은 결과 → 볼 이유 없음 | Wordle: 매일 새 문제 |
| **소속감** | 혼자 하는 서비스 → 의무감 없음 | 길드전: 내가 안 하면 팀이 진다 |
| **진행감** | 어제와 오늘이 같음 → 성장 없음 | 듀오링고: 연속 출석 스트릭 |

### 현재 제약 조건
- 유저 계정/로그인 없음
- 월렛 커넥트 없음 (읽기 전용)
- Vercel 무료~Hobby 티어 (cron 제한)
- 수익화 없음

---

## 제안 1: Weekly World Boss (변동성)

> 비유: 매주 월요일 아침, 마을 광장에 새로운 드래곤이 나타난다. 혼자서는 못 이긴다. 마을 사람들이 함께 때려야 한다.

### 컨셉
매주 "보스 지갑" 1개가 자동 선정된다. 모든 사용자가 자기 지갑으로 보스에게 배틀을 걸 수 있고, 커뮤니티 전체의 누적 데미지로 보스를 쓰러뜨린다.

### 메커니즘

```
매주 월요일 00:00 UTC
  → Vercel Cron이 "이번 주 보스" 선정
  → 보스 HP = 보스 Power × 100 (예: Power 50,000 → HP 5,000,000)
  → 보스 페이지 공개 (/boss)

사용자가 보스에게 배틀
  → 기존 배틀 엔진 그대로 사용
  → 승패 무관하게 "딜량" 기록 (보스 HP 감소량 = 내 Power × 턴 수 × 승리배율)
  → KV에 누적: boss:{weekId}:totalDamage, boss:{weekId}:participants

일요일 23:59 UTC
  → 보스 HP ≤ 0이면 → "레이드 성공"
  → 참가자 전원에게 주간 배지 부여 (localStorage + KV 기록)
  → 보스 HP > 0이면 → "레이드 실패" + 보스 다음 주 연장(HP 50%로 재등장)
```

### 돌아오는 이유
- **월요일**: "이번 주 보스 누구지?" → 호기심
- **수~목요일**: "보스 HP 얼마 남았지?" → 진행 확인
- **금~일요일**: "아직 안 때렸는데 이번 주 안에 참여해야 배지 받는다" → FOMO
- **공유 트리거**: "우리 vitalik.eth 보스 잡았다!" → 커뮤니티 달성감

### 구현 범위

| 작업 | 파일 | 난이도 |
|------|------|--------|
| 보스 선정 로직 | `lib/boss.ts` (유명 지갑 풀에서 주간 해시로 선정) | 낮음 |
| 보스 페이지 | `app/boss/page.tsx` (보스 카드 + HP바 + 참가자 수) | 중간 |
| 레이드 API | `api/boss/attack/route.ts` (기존 배틀 엔진 재사용) | 낮음 |
| 주간 Cron | `api/boss/rotate/route.ts` | 낮음 |
| 주간 배지 | KV에 `badge:{address}:{weekId}` 기록 | 낮음 |

**예상 소요: 2~3일**

## 제안 2: Class War — 8진영 주간 전쟁 (소속감)

> 비유: 월드컵. 너는 대한민국 팀을 선택한 게 아니라 태어난 것이다. 하지만 경기가 시작되면 미친 듯이 응원한다. 왜? 내가 속한 진영이니까.

### 컨셉
8개 클래스가 매주 "전쟁"을 벌인다. 모든 배틀 결과가 해당 클래스의 점수에 자동 반영된다. 주간 승리 클래스에 속한 사용자 전원이 보상을 받는다.

### 메커니즘

```
모든 배틀 결과 발생 시:
  → 승자 클래스에 +3점, 패자 클래스에 +1점 (참여 자체에 보상)
  → KV: classwar:{weekId}:{classId}:score

주간 집계:
  → 8개 클래스 순위 산출
  → 1위 클래스: "이번 주의 지배 클래스" 타이틀
  → 해당 클래스 사용자들: 다음 주 배틀에서 +5% 데미지 버프

랜딩 페이지 표시:
  → "이번 주 Class War 현황" 실시간 바 차트
  → "당신의 클래스(Hunter)는 현재 3위입니다"
  → "1위(Rogue)와 120점 차이 — 배틀 40판이면 역전 가능"
```

### 돌아오는 이유
- **부족 본능(Tribal instinct)**: "내 클래스가 지고 있다" → 배틀을 더 하러 온다
- **사회적 의무감**: "나 하나쯤이야"가 아니라 "내가 안 하면 Hunter가 진다"
- **주간 리셋**: 매주 새로운 판이 열린다 → 영구적인 신선함
- **공유 트리거**: "Hunter 1위 달성! 이번 주 우리가 이겼다" → 진영 자랑

### 핵심 설계 디테일

```typescript
// 랜딩 페이지에 표시할 Class War 위젯
{
  weekId: "2026-W10",
  standings: [
    { classId: "rogue", score: 4820, rank: 1, delta: "+340 today" },
    { classId: "hunter", score: 4700, rank: 2, delta: "+280 today" },
    // ...
  ],
  myClass: "hunter",  // 가장 최근 생성한 카드 기준 (localStorage)
  message: "Hunter는 2위입니다. 1위까지 배틀 40판!"
}
```

**예상 소요: 1~2일** (기존 배틀 API에 점수 집계 훅만 추가)

## 제안 3: Wallet Evolution — 시간 여행 카드 (진행감)

> 비유: 졸업 앨범. 10년 전 사진을 보면 웃긴다. 지금의 나와 비교하면 더 웃긴다. 그리고 다음 해에 또 열어본다.

### 컨셉
지갑의 과거 시점별 카드를 생성한다. "2020년의 당신은 Lv.3 Warrior였지만, 2023년에 Lv.28 Rogue가 되었다"를 시각적으로 보여준다.

### 메커니즘

```
/api/generate에 ?snapshot=2022 파라미터 추가
  → Alchemy API: getAssetTransfers에 toBlock 필터 적용
  → 해당 시점까지의 트랜잭션만으로 스탯/클래스 재계산
  → 결과: 연도별 카드 배열

/result/[address] 페이지에 "Timeline" 탭 추가
  → 가로 스크롤 타임라인: 2020 → 2021 → ... → 2026
  → 각 시점의 미니 카드 (클래스 아이콘 + 레벨 + Power)
  → 클래스 변경 시점 하이라이트: "2022: Warrior → Rogue로 전직!"
  → 가장 극적인 변화를 "결정적 순간"으로 표시
```

### 공유 가능한 출력물 예시

```
┌──────────────────────────────────────────┐
│  vitalik.eth의 진화                       │
│                                          │
│  2016: Lv.5 Priest    → Power 3,200      │
│  2018: Lv.18 Priest   → Power 12,400     │
│  2020: Lv.31 Elder Wizard → Power 38,000 │
│  2023: Lv.45 Elder Wizard → Power 52,000 │
│  2026: Lv.52 Elder Wizard → Power 61,000 │
│                                          │
│  "6년간 Priest에서 Elder Wizard로 진화"   │
└──────────────────────────────────────────┘
```

### 돌아오는 이유
- **자기 서사(Self-narrative)**: "나의 온체인 성장 스토리"는 1회 카드보다 공유 욕구가 훨씬 강하다
- **시간에 따른 변화**: 새 트랜잭션이 쌓이면 타임라인 끝이 바뀐다 → 분기별 재방문 동기
- **비교 욕구**: "2020년의 나 vs 2026년의 나" + "나의 성장 vs 친구의 성장"
- **공유 트리거**: 타임라인 이미지는 단일 카드보다 훨씬 흥미로운 콘텐츠다 (서사가 있으니까)

### 구현 고려사항

| 이슈 | 해법 |
|------|------|
| Alchemy API 과거 데이터 비용 | 연도별 스냅샷 캐싱 (과거는 변하지 않으므로 영구 캐시 가능) |
| API 호출 수 증가 | 최초 1회만 전체 히스토리 조회 → 연도별로 분할 계산 → KV 캐시 |
| 로딩 시간 | 현재 카드 먼저 표시 → 타임라인은 비동기 로드 |

**예상 소요: 3~4일** (Alchemy 히스토리컬 쿼리 + 타임라인 UI + 공유 이미지)

## 3개 기능의 시너지

```
                    ┌─────────────────┐
                    │   Weekly Boss   │ ← 매주 월요일 돌아옴
                    │  (변동성)        │
                    └────────┬────────┘
                             │ 배틀 결과가
                             ▼
                    ┌─────────────────┐
                    │   Class War     │ ← 내 클래스가 지고 있으면 돌아옴
                    │  (소속감)        │
                    └────────┬────────┘
                             │ 배틀이 쌓이면
                             ▼
                    ┌─────────────────┐
                    │  Evolution      │ ← 분기마다 타임라인 확인하러 돌아옴
                    │  (진행감)        │
                    └─────────────────┘
```

- **Boss가 Class War를 먹인다**: 보스 레이드 배틀도 Class War 점수에 반영 → 보스 주간에 배틀 폭증
- **Class War가 Evolution을 먹인다**: 배틀이 많아지면 랭킹이 움직이고, 랭킹 변화가 타임라인의 "이벤트"가 됨
- **Evolution이 Boss를 먹인다**: 타임라인에서 보스 레이드 참여 기록이 보임 → "빈 주"가 아쉬워서 다음 주에 참여

## 기존 제안 (비교용): Daily Duel + Evolution Unlock + Bookmark Status + Season Quest

### Daily Duel — 오늘의 대결 (Wordle 모델)
- 매일 1개의 대결이 출제된다. 두 지갑이 주어지고, 유저는 결과를 보기 전에 승자를 예측한다.
- 구현: `/daily` 페이지, 지갑 풀 50~100개, 날짜 기반 deterministic 매칭, LocalStorage 연승 기록
- 소요: 1~2일

### Evolution Unlock — 스프라이트 진화 해금
- 현재 6단계 진화 스프라이트를 서비스 내 행동으로 해금하는 시스템으로 변경
- Tier 1~6: 카드 생성 → 첫 배틀 → Daily Duel 3일 연속 → 배틀 10승 → Class War 3주 → 시즌 TOP 100
- 소요: 3~4일

### Bookmark Status Page — 내 지갑 대시보드
- `/wallet/[address]` 페이지를 "내 지갑 홈"으로 만들고, 북마크를 유도
- 내 카드 요약, 최근 배틀 이력, 랭킹 변동, Class War 기여도, 업적/진화 진행도
- 소요: 2~3일

### Seasonal Story Quest — 시즌 스토리 퀘스트
- 90일 시즌 동안 3막 구조의 스토리가 진행, 조건 달성으로 해금
- 소요: 4~5일

## 통합 우선순위 매트릭스

| 순위 | 기능 | 구현 난이도 | 리텐션 효과 | 바이럴 효과 | 추천 이유 |
|------|------|-------------|-------------|-------------|-----------|
| **1** | **Class War** | 낮음 (1~2일) | 높음 (소속감) | 중간 (클래스 자랑) | 최소 비용으로 최대 리텐션. 기존 배틀 API에 점수 집계만 추가 |
| **2** | Weekly World Boss | 중간 (2~3일) | 높음 (습관+FOMO) | 높음 (달성감 공유) | Class War와 시너지. 주간 리듬 생성 |
| **3** | Daily Duel | 낮음 (1~2일) | 높음 (습관) | 높음 (연승 공유) | LocalStorage만으로 동작. 독립적 구현 가능 |
| **4** | Bookmark Status | 낮음 (2~3일) | 중간 (확인 욕구) | 낮음 | 기존 데이터의 다른 뷰. 새 데이터 불필요 |
| **5** | Wallet Evolution | 높음 (3~4일) | 매우 높음 (서사) | 매우 높음 (공유) | 가장 임팩트 크지만 Alchemy 히스토리컬 쿼리 최적화 필요 |
| **6** | Evolution Unlock | 중간 (3~4일) | 높음 (수집) | 높음 (진화 공유) | 다른 기능 의존성 높음 (Daily Duel, Class War 선행 필요) |
| **7** | Season Quest | 높음 (4~5일) | 매우 높음 (서사) | 중간 | 스토리 콘텐츠 작성 비용 높음. 리텐션 기반 확보 후 추가 |

## 실행 원칙

> **코드 전에 검증.**
>
> Class War 하나만 추가해도 "매주 돌아올 이유"가 생긴다. 1~2일 구현이고 기존 배틀 API 수정만으로 동작한다.
> 이것을 먼저 배포하고, 실제로 주간 배틀 수가 변하는지 1주일 관찰하라.
>
> 전부 만들지 마라. **Class War → 데이터 확인 → Weekly Boss → 데이터 확인** 순서로 한 번에 하나씩, 측정하면서 진행하라.
