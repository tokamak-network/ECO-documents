# "On-Chain Personality Test" 리프레이밍 제안

> **결론**: 현재 "지갑 RPG"라는 기술 중심 포지셔닝이 세그먼트 B 유저에게 "내 대상이 아니다"라는 심리적 자기검열을 유발할 수 있다. "온체인 성격 테스트"로 리프레이밍하면 이 허들을 낮출 가능성이 있다. **카피 변경은 1시간이면 적용 가능하고 즉시 롤백할 수 있으므로, 최소 비용 실험으로 가설을 먼저 검증할 것을 제안한다.**
>
> **출처**: IH 댓글 (Rachealbuildgrowth, 2026-03-01)
> **상태**: 연구/검토 단계 — 구현 미확정

## 대상 유저 세그먼트 정의

| 세그먼트 | 정의 | 현재 도달 여부 |
|----------|------|---------------|
| **A. 크립토 네이티브** | 지갑 보유 + DEX/DeFi 경험자. 주소를 외우거나 즉시 복사 가능 | 부분 도달 (IH, PH 일부) |
| **B. 크립토 보유자** | 지갑 보유 + HODLer. 주소 찾는 데 1-2분 소요 | 미도달 |
| **C. 일반 유저** | 지갑 미보유. "0x"가 뭔지 모름 | 미도달 |

이 문서의 핵심 타겟은 **세그먼트 B**다. 지갑은 있지만 주소 입력이라는 행위가 심리적 허들인 유저.

## 원본 피드백

> **요약**: 현재는 "크립토 정체성을 이해한 뒤 재미를 경험"하는 순서인데, "호기심/재미가 먼저, 크립토 맥락은 나중"으로 바꾸면 진입 동기가 강해진다. "온체인 성격 테스트"처럼 호기심이 첫 상호작용을 이끌고, 배틀은 정체성 발견 이후에 오는 구조.

## 핵심 인사이트

### 1. 동기 순서 (Order of Motivation) 문제

현재 흐름:
```
"크립토 지갑이 뭔지 이해" → "주소 입력" → "재미 경험"
```

제안 흐름:
```
"호기심/재미" → "결과 확인" → "크립토 맥락 이해"
```

**비유**: 현재는 "요리 재료를 먼저 설명하고 맛보게 하는" 순서. 제안은 "먼저 맛보게 하고 재료를 설명하는" 순서.

### 2. "Personality Test" 프레이밍

- 현재 포지셔닝: "RPG character generator for Ethereum wallets" — **기술 중심**
- 제안 포지셔닝: "On-chain personality test" — **호기심 중심**
- BuzzFeed 퀴즈, 16 Personalities(MBTI) 같은 "나는 어떤 유형?" 심리가 진입 동기

**인과 가설**: 현재 카피("What Kind of Hero Is Your Wallet?")는 "지갑을 잘 아는 사람용 도구"라는 인상을 줘서 세그먼트 B가 자기 대상이 아니라고 판단하고 이탈할 수 있다. 리프레이밍은 이 **심리적 자기검열**을 줄여 시도 의지를 높일 수 있다. (대안 가설: 주소 접근 불편, 또는 단순 관심 부족)

**한계**: 리프레이밍은 "이걸 해볼 만한 것 같다"는 동기 부여에만 작용한다. 주소를 찾아 복사-붙여넣기하는 **물리적 마찰**(MetaMask 열기, 주소 복사 등)은 리프레이밍으로 해결되지 않는다. 이 문제는 별도 대응(ENS 검색 자동완성, WalletConnect 연동 등)이 필요하며, 이 문서의 범위 밖이다.

### 3. 데이터 근거 (4일 누적 분석 로그 기반)

> 표본이 소량(총 외부 방문 41-49명, 카드 생성 5건)이므로 **방향성 참고용**이다. 통계적 유의성은 없으며, A/B 테스트로 검증이 필요하다.

| 관찰 | 수치 | 시사점 |
|------|------|--------|
| wallet chips가 제품 체험 진입점 역할 | 클릭 2건 → 타인 카드 조회 2건 (100% 전환) | "남의 카드 구경"은 마찰이 극히 낮음 |
| 자기 지갑 직접 입력은 여전히 높은 허들 | Day 1: 17명 방문 → 2-3명 입력 (~15%) | 주소 입력 자체가 핵심 마찰 지점 |
| wallet chips 체험 → 자기 지갑 입력 전환은 미관측 | Day 4 유저: 타인 카드 2장 조회 후 이탈 (자기 지갑 입력 0건) | "구경"과 "자기 것 만들기"는 별개의 퍼널 단계 (N=1, 가설 수준) |
| 유일한 자발적 공유가 배틀 결과 | Day 1-4 누적 공유 1건 (배틀 결과 Twitter 공유, Day 2) | 경쟁/비교가 공유 동기일 가능성 (N=1, 가설 수준) |

> **주의**: wallet chips 전환율(100%)과 직접 입력 전환율(15%)은 **서로 다른 행동**을 측정한다. 전자는 "타인 카드 조회", 후자는 "자기 카드 생성"이다. 두 수치를 직접 비교하면 안 된다.

## 가능한 실험 방향

### A. 랜딩 카피 변경 (최소 공수)

현재: "What Kind of Hero Is Your Wallet?"
실험: "What's Your On-Chain Class?" 또는 "Discover Your Ethereum Identity"

- RPG 용어 → 성격 유형 용어로 전환
- 기술적 진입 장벽 낮춤
- A/B 테스트 가능

### B. 탐색 우선(Browse First) 진입 흐름 (중간 공수)

```
랜딩: 유명 지갑 카드 갤러리 (구경)
  → "See yours" CTA
  → 주소 입력
  → 카드 생성
```

- famous wallet chips를 갤러리/카드 프리뷰로 확장
- "구경 → 자기 것 만들기" 자연스러운 전환
- Day 4에서 wallet chips 클릭 → 타인 카드 조회 전환은 관측됨 (N=2). 단, 타인 카드 조회 → 자기 지갑 입력 전환은 미관측 (N=1). B안의 핵심 가정인 "구경 → 자기 것 만들기" 전환이 실제로 발생하는지가 검증 대상

### C. 퀴즈형 진입 (높은 공수, v2.0 이후 검토)

```
"Are you a Hunter, Guardian, or Wizard?"
→ 3-5개 자기 보고 방식 질문
  (예: "일주일에 토큰 스왑을 몇 번 하시나요?", "NFT를 수집하는 편인가요?")
→ 결과: "You're most likely a [Class]"
→ "Verify with your real wallet" CTA
→ 주소 입력 → 실제 카드 생성
```

- 지갑 주소 없이도 첫 경험 가능 (세그먼트 C까지 커버)
- 실제 결과와 예측 비교가 두 번째 동기
- 주의: 퀴즈만 하고 지갑 입력 없이 이탈할 가능성 있음
- 공수 높음 + 퀴즈 설계 필요

### D. 공유 최적화: 비교 카드 (중간 공수)

현재 공유: 개인 카드 이미지
실험: "vitalik.eth vs [나] — Hunter vs Guardian" 비교 카드

- 유일한 자발적 공유가 배틀 결과였음 (N=1이지만, 공유 동기가 비교/경쟁일 가능성 시사)
- 배틀 결과 공유 이미지 최적화가 선행 과제

## 우선순위 제안

공수(투입 시간)와 기대 임팩트(전환율 개선 기여도)를 기준으로 정렬. 각 실험은 이전 실험의 결과에 따라 착수 여부를 결정한다.

| 순위 | 안 | 공수 | 기대 임팩트 | 리스크 | 판단 근거 | 전환 조건 (→ 다음 안 착수 시점) |
|------|-----|------|-----------|--------|-----------|-------------------------------|
| 1 | **A. 카피 변경** | 1시간 | 낮음~중간 | 없음 (롤백 즉시 가능) | 공수 대비 리스크 제로. 가장 빠른 실험 | 적용 2주 후 또는 누적 방문 100명 도달 중 먼저 오는 시점에 리뷰. 자기 지갑 입력 +1건 이상이면 B 착수. 0건이면 A 롤백 후 B를 독립 실험으로 착수 |
| 2 | **B. 탐색 우선** | 1-2일 | 중간 | 중간 ("구경만 하고 이탈" 악화 가능) | wallet chips 체험 진입점 역할 관측됨. 갤러리 확장은 이 패턴의 자연스러운 확장 | 적용 2주 후 리뷰. 자기 지갑 입력 비율이 A안 대비 하락하면 롤백 |
| 3 | **D. 비교 공유** | 1-2일 | 중간 | 낮음 | 자발적 공유 1건이 배틀 결과. 공유 이미지 개선은 바이럴 루프 강화 | B와 병행 가능. 공유 클릭 수 추적 |
| 4 | **C. 퀴즈형** | 1-2주 | 높음 (세그먼트 C 커버) | 높음 (퀴즈에서 끝날 수 있음) | 공수 대비 불확실성 높음. v2.0 이후 검토 | A/B 실험 완료 후 전환율 데이터 기반 판단 |

## 미결 질문 + 검증 계획

| 질문 | 검증 방법 | 담당 | 성공 기준 |
|------|----------|------|----------|
| 카피 변경이 자기 지갑 입력 행동에 영향을 주는가? | **정량**: A안 적용 후 누적 외부 방문 100명 도달 시점까지 자기 지갑 입력 건수 추적. **정성**: 지인 DM 10명에게 변경 전/후 카피 2종 보여주고 "해보겠냐" 응답 수집 | Growth | 정량: 자기 지갑 입력 +1건 이상 (현재 4일간 0건). 정성: 10명 중 5명 이상 "해보겠다" |
| "on-chain personality test"가 세그먼트 B에게도 통하는가? | 지인 DM 중 세그먼트 B 해당자(지갑 보유, DeFi 미경험) 5명 별도 추적 | Growth | 5명 중 3명 이상 카드 생성 |
| 카드 갤러리가 "구경만 하고 이탈"을 악화시키지는 않는가? | B안 적용 시 자기 지갑 입력 비율 추적 (A안 기간 대비) | Frontend + Growth | 자기 지갑 입력 비율이 A안 기간 대비 하락하지 않을 것 |
| 퀴즈형 진입이 실제 지갑 입력 전환율을 높이는가? | C안은 v2.0 이후. 사전 검증으로 Typeform 프로토타입 → 지인 10명 테스트 | Growth | 퀴즈 완료자 중 50% 이상이 "지갑으로 확인하기" 클릭 |
