## Introduction

As Tokamak Network evolves towards complete decentralization, we are proud to announce a revolutionary step in DAO governance: the launch of our Community Version for Tokamak DAO. This transition represents more than just a technical upgrade—it embodies the true spirit of a decentralized autonomous organization where governance power genuinely resides with the community.

Our current DAO interface, while functional, still relies on centralized infrastructure that contradicts the fundamental principles of autonomous governance. By empowering our community to independently operate and host the DAO service, we are removing the last barriers to true self-governance and ensuring that Tokamak DAO becomes genuinely autonomous and community-controlled.

## Introducing the Community Version

### Innovation Towards True Decentralization

The **Community Version** is a new approach developed to fundamentally solve the centralization issues of the existing DAO service.

**Limitations of the Existing Service:**
The existing Tokamak DAO service was completely dependent on backend services such as subgraphs and APIs. Without this backend infrastructure, service operation was impossible, resulting in the service inevitably operating in a centralized state.

**Innovation of the Community Version:**
The community version has completely eliminated this backend dependency. No longer relying on centralized backend infrastructure, **anyone can now independently operate the DAO service**.

**Core Features:**

- **Backend Independence**: No need for centralized backend services like subgraphs and APIs
- **Complete Autonomy**: Individual users or communities can operate services independently
- **True Decentralization**: No longer dependent on a single operating entity
- **Identical Functionality**: Provides essentially the same DAO features as the existing service

## Background and Conditions of Transition

### Simplified Transition Strategy

The community version fundamentally provides the same functionality as the existing DAO service, but with the key difference of independence from backend infrastructure.

**Parallel Operation Phase:**
Tokamak Network will run both the existing DAO service and the community version simultaneously for 2 weeks, providing users with sufficient time to familiarize themselves with the new environment and ensure seamless migration.

**Community-Only Phase:**
After the 2-week parallel operation period, the existing DAO service will be discontinued, and only the community version will be officially hosted by Tokamak Network for an additional 2 weeks.

**Complete Transition Phase:**
After 4 weeks total, the service will transition fully to community or individual user hosting, including local environments.

**Exclusive Local Environment Features:**

When running the DAO Community Version locally, users gain access to advanced governance tools not available in hosted services:

- **Agenda Simulation Node**: A powerful validation feature that allows proposal creators to test and verify their agendas before submission. This simulation environment ensures that proposed governance actions will execute correctly, preventing failed proposals and protecting the DAO from potential execution errors.

*Note: The Agenda Simulation Node is exclusively available for local deployments and will not be supported in hosted versions of the Community Version.*
